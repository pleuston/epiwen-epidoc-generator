Epiwen · EpiDoc generator for Chinese stone inscriptions

Epiwen is a client-side, bilingual (EN / 中), dependency-free web tool that produces valid EpiDoc / TEI records for the stone sutras project. Developed for the Epiwen encoding project; modelled on the Hamburg Metadata Generator for EpiDoc.

Nothing is uploaded — the XML is built live in the browser. Records are contributed via GitHub pull requests and reviewed before they appear in the catalog.

Data model

The platform is organized around nine entity types and the relationships between them. Hover a node to highlight its connections.

Features

  • Bilingual UI — toggle EN / 中 / EN+中; every field is labelled in both languages.
  • Project vocabularies — material, object type (the five carriers: 摩崖 / 碑 / 經幢 / 窟壁 / 題記), script, language, licence, and a list of canonical sutras with CBETA/Taishō anchors. Controlled vocabulary is editable in vocab.js.
  • Reign-era date helper — pick a 年號 and a reign year; the tool computes the exact @when and fills @notBefore / @notAfter / @calendar / @datingMethod.
  • Several texts on one object — Add text repeats the block so one support bearing several texts (e.g. 碑陽 sutra + 碑陰 colophon) yields one msDesc with multiple <msItem> and a <div type="edition"> holding one <div type="textpart"> per text.
  • Live preview + well-formedness check; Copy and Download .xml.
  • Propose to GitHub — one-click contribution flow: copies the XML and opens GitHub's web editor at records/your-file.xml. GitHub handles the fork and pull request automatically.

Catalog, comparing rubbings & favorites

  • Selectable entries — click a catalog row (or focus it and press Enter/Space) to open its preview. The row itself is the control — there is no separate Preview button. Copy XML and Edit stay as buttons and don't trigger the row.
  • Site filter — on the Objects and Inscriptions tabs, a Site 遗址 dropdown narrows the list to the inscriptions belonging to one site (built from each record's <origPlace>, falling back to <repository>).
  • Two rubbing views — a toggle on the Rubbings tab: Every rubbing (flat, one row per manifest) or Compact (by inscription), which draws the different collections together under the same inscription (grouped by surrogateOf).
  • IIIF comparison viewer (viewer.html, Mirador) — add rubbings to the floating comparison bar, then Open side by side; each manifest opens as a pane in a mosaic for character-by-character comparison.
  • Detach — ⧉ Detach pops the viewer out of the catalog window into a separate, freely-resizable window. Available on the comparison bar, inside the viewer, and on every saved favorite.
  • Favorites (favorites.html) — ★ Save to favorites stores the current comparison (a named set of manifests) per-browser in localStorage. The Favorites page re-opens, detaches, or deletes each saved comparison. Personal and offline — no server round-trip.

Scope

The tool covers the teiHeader + physical description + edition skeleton — the scope of Epiwen session 2 (Headers & physical description). It does not model the per-glyph measured layer (geometry + condition grade); that is documented separately and is out of scope for any header generator.

Roadmap — dynamic version

This is the static "necessary" tier of the Epiwen tooling. The desirable next step is a dynamic version with stored records, structured input fields, validation, and search — on the model of EDEP / EFES / an eXist-db data-entry UI. The serializer (generator.js) is a pure function with no DOM dependency, making that migration straightforward.
