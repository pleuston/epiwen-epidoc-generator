Getting Involved

Epiwen uses GitHub — a free, web-based platform — to coordinate scholarly contributions: research discussions, data questions, technical feedback, and encoded records. You do not need programming experience to participate. This guide walks you through the full process from account creation to opening your first issue.

What contributors can do

RoleWhat you can doAccess needed
Participant Open and comment on issues — research questions, interpretive debates, technical feedback GitHub account + project invitation
Editor All of the above, plus contribute encoded EpiDoc records via the editor tool GitHub account + editor invitation + personal access token

Step 2 — Request access to the project

The Epiwen platform is a private working environment for the Buddhist Stone Sutras research group. To contribute, you need to be invited by a project administrator.

1
Contact the project team

Send your GitHub username (not your password — only the username) to the project coordinator. You can find your username at the top right of any GitHub page after signing in, or under Settings → Profile.

2
Accept the invitation email

GitHub sends an invitation to your email address. Open it and click Accept invitation. You can also accept via the notification bell (🔔) on GitHub after signing in.

On the right sidebar, click Labels and choose the appropriate category:

How to write a good issue

A well-written issue gets a faster, more useful response.

discussion Research & conceptual

  • Identify the inscription by its ID or conventional name (e.g. BSY_Diamond, JSY_01)
  • State your question or position clearly in the title
  • Cite the relevant scholarship: author (year), page
  • Quote the primary source passage if applicable
  • Distinguish between what the source says and your interpretation
  • It's fine to leave a question open — the discussion thread resolves it

technical Technical & data

  • Describe what you expected to see and what you actually saw
  • Name the page or record involved (e.g. Map page, record BSS_001.xml)
  • Note your browser and device if relevant (e.g. Safari on iPhone)
  • One problem per issue — don't bundle multiple bugs together
  • For data errors, include the correct value and its source
  • Screenshots are welcome — paste them directly into the GitHub editor
Example: good discussion title "JSY_01: Diamond Sūtra or Perfection of Wisdom in 8,000 Lines? — opening sequence ambiguous"
vs. a vague title: "Question about JSY"
Example: good technical title "Map page: OSGeo atlas tiles not loading on Firefox — blank grey background"
vs. a vague title: "Map broken"

What happens after you submit

  • All project members receive a notification (email and/or GitHub bell).
  • Anyone can comment on the thread — discussion continues until the question is resolved.
  • The project team may apply additional labels, ask for clarification, or link related issues.
  • When an issue is resolved it is closed — closed issues remain permanently searchable in the GitHub archive.
  • You can subscribe to an issue you didn't open by clicking Subscribe in the right sidebar to get notifications.

Becoming an editor (contributing records)

Editors can contribute encoded EpiDoc records directly via the Epiwen editor. This requires a GitHub personal access token (PAT) with repo scope, stored locally in your browser. A separate guide covers the editor workflow — click Open workflow guide → on the Docs page.

To become an editor Contact the project coordinator with your GitHub username and a brief description of your role in the project. Editors are invited individually.
